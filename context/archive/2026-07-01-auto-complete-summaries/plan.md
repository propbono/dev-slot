# Auto-Complete & Summaries — Implementation Plan

## Overview

Track consecutive strong evaluations per challenge. After 3 consecutive "strong" answers, auto-complete the challenge with a comprehensive summary (quality, confidence, strengths, improvement areas) generated by a new LLM call synthesizing the full conversation.

## Current State

- evaluate.ts stores quality/confidence/rationale in message metadata after each evaluation
- Challenges table has status (active|completed) and summary (JSONB)
- Session page shows completion banner for completed sessions, not per-challenge
- No consecutive strong tracking

## Desired End State

After each evaluation: if the answer was strong and the previous 2 answers for this challenge were also strong → autocomplete. Summary generated via `generateSummary()`, stored in challenges.summary, challenge status → completed. Session page shows completion banner only for the completed challenge, and "New Challenge" button appears.

## Implementation Approach

Single phase. evaluate.ts gets the consecutive-tracking logic. ai.ts gets `generateSummary()`. Session page gets per-challenge completion UI.

## Phase 1: Tracking + Summary + UI

### Changes Required:

#### 1. evaluate.ts — consecutive strong tracking

**File**: `src/pages/api/interview/evaluate.ts`

**Intent**: After evaluation, if quality is "strong", query the last 2 user messages for this challenge and check if they were also strong. If 3 consecutive strong → trigger summary generation and auto-complete.

**Contract**: After storing evaluation metadata:
- If quality !== "strong" → skip, redirect normally
- If quality === "strong":
  - Query `SELECT metadata, role FROM session_messages WHERE challenge_id = X AND role = 'user' ORDER BY created_at DESC LIMIT 3`
  - Check if all 3 (current + previous 2) have metadata.quality === "strong"
  - If yes: call `generateSummary()`, update challenge (status='completed', summary), redirect
  - If no: redirect normally

#### 2. generateSummary() — conversation synthesis

**File**: `src/lib/ai.ts` (new function)

**Intent**: Synthesize the full conversation (system message, challenge, all user answers, all interviewer responses) into a comprehensive summary.

**Contract**:
```ts
export interface ChallengeSummary {
  quality: "strong" | "mixed";
  confidence: number;
  rationale: string;
  strengths: string[];
  improvement_areas: string[];
}

export async function generateSummary(
  conversation: string
): Promise<ChallengeSummary>
```

Prompt: "You are evaluating a completed interview challenge. Given the full conversation below, provide a comprehensive summary. Return JSON: { quality: 'strong'|'mixed', confidence: 0-1, rationale: '...', strengths: [...], improvement_areas: [...] }"

#### 3. Session page — per-challenge completion

**File**: `src/pages/interview/[sessionId].astro`

**Intent**: When the active challenge is completed, hide the editor and show a "Challenge Complete" card with the summary + "New Challenge" button.

**Contract**:
- Query challenges table for active challenge status
- If active challenge status is "completed":
  - Hide AnswerEditor
  - Show completion card: quality badge, confidence bar, strengths list, improvement areas list
  - Show "New Challenge" button (placeholder for S-07 — generates new challenge)
- If no challenges or all completed: show "New Challenge" to start

### Success Criteria:

- Build, typecheck, lint, tests pass
- 3 consecutive strong → challenge auto-completes with summary
- Summary includes strengths + improvement areas
- Completion card visible on session page after auto-complete
